# 更新履歴（updates.json）の運用メモ

`updates.json` は更新履歴の**単一ソース**です。ここに1エントリ追記するだけで、次の3か所すべてに反映されます。

| 反映先 | 何が出るか |
| --- | --- |
| LPの更新帯（`index.html` のコックピットセクション） | 最新3件 |
| 更新履歴ページ（`updates.html`） | 全件（年ごと・新しい順） |
| アプリの青い更新バナー（`app.html`「更新内容」リンク） | 最新エントリ（ポップアップ・最大5件） |

LP・updates.html・app.html はどれも `/updates.json` を**ネットから直接取得**します（Service Worker のキャッシュ対象外。`sw.js` の `NO_CACHE_PATH_KEYWORDS` に登録済み）。なので追記＝即反映で、`CACHE_VERSION` の bump は不要です。

---

## 追記手順（リリースのたびに）

本番マージのとき、`updates.json` の**配列の先頭**に1エントリ足すだけです。並びは「新しい順」を保ってください。

```json
[
  {
    "date": "2026-06-12",
    "tag": "feature",
    "summary": "案件一覧に絞り込み検索を追加しました"
  },
  ...既存のエントリ...
]
```

### フィールド

| フィールド | 必須 | 内容 |
| --- | --- | --- |
| `date` | ○ | `YYYY-MM-DD`（例 `2026-06-12`）。表示時に自動で `2026.06.12` に整形されます |
| `tag` | ○ | `fix` / `feature` / `improve` の3種のみ（下表） |
| `summary` | ○ | ユーザー向けの一行サマリー。**です・ます調／開発用語を使わない** |
| `version` | 任意 | SWのバージョン文字列など。今は未使用（付けても表示はされません） |

### tag の3種

| `tag` | 表示ラベル | 色 | 使う場面 |
| --- | --- | --- | --- |
| `fix` | 改修 | 青 | 不具合の修正 |
| `feature` | 新機能 | ゴールド | 新しくできることが増えた |
| `improve` | 改善 | 緑 | 既存機能の使い勝手・速度・安全性の向上 |

### summary の書き方（トーンの見本）

- ✅「見積書・請求書の品目欄で、数量2以上のときに単価と金額が正しく表示されない問題を修正しました」
- ✅「クライアント詳細に、オンライン申請の形式へ整形してコピーできるボタンを追加しました」
- ❌「invoiceのqty>1でunit_priceの二重計算をfix」← 開発用語・体言止めはNG

> ユーザー（行政書士の先生）が読んで「何が、どう良くなったか」が一読で分かる文にします。

---

## 注意

- **JSONの文法に注意**：末尾カンマ禁止、文字列は `"` で囲む。追記後はエディタの赤線が出ていないか確認。
- 取得に失敗しても（オフライン等）、各画面は静的フォールバック（HTMLに直書きの最新3件）を表示するか、リンクを出さない安全側に倒れます。バナーの再読み込み機能は影響を受けません。
- `app.html` 自体を変えていない更新（`updates.json` だけの追記）では `sw.js` の `CACHE_VERSION` は**上げません**。
